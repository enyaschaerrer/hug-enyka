## Plateforme technique et outils

Une plateforme technique et des outils de developpement doivent etre mis en oeuvre via l'exploitation des logiciels suivants :

- Git doit etre utilise pour la gestion des codes sources de l'application
- Laravel pour le framework de developpement back-end
- Vue.js pour le developpement des parties reactives front-end
- Les etudiants ont libre choix pour les frameworks UI
- MySQL pour le systeme de gestion de base de donnees
- Infomaniak pour l'hebergement et le deploiement

## Criteres techniques

Projet fonctionnel de bout en bout

- Code lisible et structure (eviter les generations IA non maitrisees)
- Architecture coherente du code (modulaire, non-redondant, etc.)
- Vues structurees avec une approche par composants (pour une organisation coherente du CSS)
- Les principes de securite Web de base doivent etre respectes (validation des entrees, gestion des acces, etc.)
- Messages d'erreur propres
- Les bonnes pratiques Git doivent etre suivies (pas de donnees sensibles sur le repo, utilisation de branches, etc.)
- Un environnement local est disponible et les procedures pour le mettre en place sont fournies
- Une procedure de mise en production de l'application doit etre fournie
- Un jeu de donnees de test doit etre fourni (seeder)
